# Prompt 3 — Evolução conceitual do convite web

Jarvis, analise a versão atual publicada do convite e evolua a página para uma experiência visual mais conceitual, emocional, fluida e memorável.

A página atual ficou estruturalmente correta, mas ainda parece uma página web simples, escura e estática, próxima de um template de WordPress ou landing page genérica. O objetivo agora é transformar o convite em uma experiência digital de casamento íntimo, com identidade própria, movimento, emoção, profundidade visual e aparência artesanal/premium, sem parecer feito por IA.

O convite continuará sendo enviado pelo WhatsApp, então a prioridade absoluta é a experiência mobile.

---

## Diagnóstico da versão atual

A versão atual tem uma boa base de conteúdo, mas não entrega ainda o sentimento esperado.

Problemas percebidos:

- Está dark e cinza demais.
- O visual está frio e pouco acolhedor.
- A página parece dividida em blocos/seções muito duras.
- Falta fluidez na transição entre as seções.
- Falta movimento real.
- Falta uso emocional das fotos.
- Falta profundidade no background.
- Falta glowmorphism perceptível, mas elegante.
- Falta aparência de convite de casamento.
- A fonte atual não transmite o tom ideal.
- A experiência parece mais informativa do que afetiva.
- A página parece simples demais para a proposta original.

O objetivo não é criar algo luxuoso ou exagerado. O objetivo é criar um convite íntimo, moderno, elegante, humano e afetivo.

A página precisa sair de:

> uma landing page dark com textos sobre casamento

para:

> uma experiência digital íntima, fluida e emocional de casamento.

---

## Entendimento principal

Este não é um site comercial.

Este não é um template pronto.

Este não é uma landing page institucional.

Este é um convite web de casamento para Douglas e Regiane, feito para pessoas próximas, que fazem parte da história do casal.

A pessoa que abrir o link pelo celular precisa sentir:

> Recebi esse convite porque sou importante para eles.

A experiência precisa comunicar:

- amor;
- família;
- fé;
- intimidade;
- simplicidade;
- carinho;
- memória;
- presença;
- celebração;
- verdade.

## Direção visual revisada

Manter a estética dark, mas aquecer o visual.

A versão atual está muito preta/cinza. O novo visual precisa continuar elegante, mas com mais calor emocional.

### Paleta recomendada

Usar como base:

- preto suave;
- grafite quente;
- cinza escuro;
- cinza fosco;
- off-white;
- champagne suave;
- dourado queimado discreto;
- marrom/cobre muito sutil em glows;
- transparências com blur.

Evitar:

- preto puro em excesso;
- cinza chapado;
- branco forte demais;
- dourado muito luxuoso;
- rosa chamativo;
- efeitos carnavalescos;
- aparência de festa grande.

O resultado precisa parecer dark afetivo, não dark corporativo.

---

## Background e atmosfera

Substituir o fundo estático e pesado por um background vivo e cinematográfico.

Implementar:

- gradientes radiais suaves;
- halos de luz em champagne/cobre com baixa opacidade;
- textura fina de ruído/grain;
- light leaks discretos;
- camadas com blur;
- profundidade visual;
- movimento quase imperceptível no background;
- transições suaves entre áreas.

O fundo deve acompanhar a narrativa da página, não parecer apenas uma textura repetida.

Sugestão de composição:

- Hero com foto escurecida e gradiente.
- Seções intermediárias com halos suaves.
- Área de história com atmosfera mais acolhedora.
- Área de cerimônia com luz mais serena.
- Encerramento com fundo emocional e limpo.

---

## Fotos reais

Já existem 3 fotos na pasta do projeto. Elas precisam ser usadas de verdade.

Não manter placeholders cinzas.

As fotos devem fazer parte da narrativa, não serem apenas uma galeria simples.

### Uso obrigatório das imagens

1. **Hero**
   - Usar uma das fotos como fundo emocional.
   - Aplicar overlay escuro.
   - Aplicar blur leve ou escurecimento elegante.
   - Manter boa leitura dos textos.
   - Adicionar glow suave atrás dos nomes.

2. **Seção de memórias**
   - Exibir as 3 fotos reais.
   - Criar composição orgânica, não apenas 3 cards iguais sem alma.
   - Desktop: composição assimétrica elegante.
   - Mobile: empilhar imagens com respiro e bom enquadramento.

3. **Encerramento**
   - Usar uma foto ou recorte visual de forma quase imperceptível no fundo.
   - A foto deve aparecer como memória, não como destaque agressivo.

### Tratamento das fotos

Aplicar:

- object-cover;
- bordas arredondadas;
- overlay escuro suave;
- leve saturação quente;
- sombra macia;
- borda champagne quase invisível;
- hover/tap com leve clareamento;
- scale mínimo;
- motion de entrada.

As fotos devem transmitir carinho e memória, sem transformar a página em álbum.

## Animações e fluidez

A página atual está estática. Implementar movimento suave com Framer Motion.

O movimento deve ser elegante, lento e emocional. Nada exagerado.

### Implementar

- entrada do Hero em camadas;
- fade + slide nas seções;
- stagger nos cards;
- parallax leve nas fotos;
- glow respirando lentamente;
- linha da timeline animando conforme aparece;
- fotos com scale suave ao entrar;
- cards com hover/tap delicado;
- transições mais orgânicas entre seções;
- scroll indicator com movimento leve;
- background com movimento sutil.

### Evitar

- animação rápida demais;
- efeitos pulando;
- excesso de particles;
- 3D pesado;
- animações que prejudiquem mobile;
- travamentos;
- sensação de site chamativo.

O convite precisa parecer vivo, não barulhento.

---

## 3D e profundidade visual

Adicionar sensação de profundidade, mas sem pesar a página.

Pode usar:

- camadas flutuantes com blur;
- cards com perspectiva sutil;
- elementos decorativos leves;
- halos em movimento;
- parallax por scroll;
- pequenos detalhes 3D via CSS.

Não usar 3D pesado ou WebGL complexo se comprometer a performance mobile.

O foco é profundidade visual e fluidez, não espetáculo tecnológico.

---

## Glassmorphism e glowmorphism

A versão atual tem cards escuros, mas ainda não entrega bem glass/glow.

Atualizar os cards para terem mais refinamento.

### Direção de card

Usar:

```css
background: rgba(255, 255, 255, 0.055);
backdrop-filter: blur(18px);
border: 1px solid rgba(255, 255, 255, 0.12);
box-shadow:
  0 20px 60px rgba(0, 0, 0, 0.35),
  inset 0 1px 0 rgba(255, 255, 255, 0.08);
```

### Glow sutil

Usar em pontos específicos:

```css
box-shadow:
  0 0 0 1px rgba(212, 175, 112, 0.14),
  0 0 40px rgba(212, 175, 112, 0.08);
```

Não colocar glow em tudo. Usar como detalhe para conduzir o olhar.

## Responsividade mobile

A página será enviada via WhatsApp e provavelmente aberta em tela pequena.

Garantir:

- hero impactante no mobile;
- textos legíveis;
- parágrafos curtos;
- espaçamento confortável;
- fotos bem enquadradas;
- cards empilhados;
- timeline vertical;
- botões tocáveis;
- animações leves;
- performance boa;
- carregamento rápido;
- nada pesado demais;
- contraste adequado;
- experiência mobile melhor que desktop.

No desktop, a página pode ter composições mais largas e assimétricas. No mobile, tudo precisa ser simples, bonito e fluido.

## Performance

Não comprometer a performance.

Regras:

- otimizar imagens;
- evitar vídeos pesados;
- evitar WebGL pesado;
- limitar partículas;
- usar animações CSS/Framer Motion leves;
- respeitar `prefers-reduced-motion`;
- lazy load nas imagens abaixo da dobra;
- manter layout estável sem shifts.
